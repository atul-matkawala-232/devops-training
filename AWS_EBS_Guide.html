Section 01

What is EBS?

Amazon Elastic Block Store (EBS) is a block-level storage service designed to work with Amazon EC2 (virtual servers). Think of it as a high-speed external hard drive that you attach to your cloud server.

Real-world analogy: Imagine your EC2 instance is a laptop. EBS is the hard drive inside that laptop. Without the hard drive, the laptop can't store your OS, files, or data — and EBS plays that exact role for EC2.

Key Characteristics

🔗
Network-Attached

EBS volumes connect to EC2 over the network, not physically — making them flexible.

♻️
Persistent

Data survives EC2 restarts and shutdowns — unlike instance store (ephemeral) storage.

📐
Scalable

Resize volumes on the fly from 1 GiB up to 64 TiB without stopping your instance.

📍
AZ-Bound

Each EBS volume lives in a specific Availability Zone — same as the EC2 it attaches to.

Section 02

How It Works

EBS uses a block storage model. Unlike file storage (folders/files), block storage works at a raw disk level — operating systems, databases, and applications interact with it as if it were a physical disk.

Block Storage vs. Other Types

TypeExamplesBest ForAnalogy
Block StorageEBS, SANDatabases, OS disksHard drive
File StorageEFS, NFSShared files, contentNetwork folder
Object StorageS3Backups, media, static filesFiling cabinet

Attaching an EBS Volume — CLI Example

AWS CLI 
# Step 1: Create a new 20 GB gp3 volume in us-east-1a
aws ec2 create-volume \
  --size 20 \
  --volume-type gp3 \
  --availability-zone us-east-1a \
  --tag-specifications 'ResourceType=volume,Tags=[{Key=Name,Value=MyAppDisk}]'

# Step 2: Attach the volume to an EC2 instance
aws ec2 attach-volume \
  --volume-id vol-0abcd1234efgh5678 \
  --instance-id i-0123456789abcdef0 \
  --device /dev/xvdf

# Step 3: SSH into your instance and format + mount it
sudo mkfs -t ext4 /dev/xvdf
sudo mkdir /data
sudo mount /dev/xvdf /data
echo "/dev/xvdf /data ext4 defaults,nofail 0 2" | sudo tee -a /etc/fstab
ℹ️

One-to-One Rule: A single EBS volume can only be attached to one EC2 instance at a time (in the same AZ). Exception: EBS Multi-Attach (only for io1/io2 volumes) allows attaching to multiple instances simultaneously.

Section 03

Volume Types

EBS offers different volume types tuned for different workloads — from budget-friendly HDDs to blazing-fast NVMe SSDs.

⚡
gp3 (General Purpose SSD)

Best for most workloads. 3,000–16,000 IOPS. Cost-effective. Default choice.

SSD
⚡
gp2 (General Purpose SSD)

Older general-purpose SSD. IOPS tied to volume size (3 IOPS/GiB). Being phased out in favour of gp3.

SSD
🚀
io2 Block Express

Highest performance SSD. Up to 256,000 IOPS. For mission-critical databases (Oracle, SAP).

HIGH PERF
💿
st1 (Throughput HDD)

Low cost HDD for big data workloads. High throughput, not high IOPS (MapReduce, Kafka logs).

HDD
🗜️
sc1 (Cold HDD)

Cheapest EBS option. Infrequently accessed data — archives, backups, cold storage.

Section 04

Volume Lifecycle

Understanding how EBS volumes are created, used, resized, and deleted is essential for managing costs and data safely.

1
Create

Choose type, size, AZ. Volume moves to "available" state. You pay from this moment.

2
Attach

Attach to an EC2 instance in the same AZ. Volume appears as a block device (e.g. /dev/xvdf).

3
Format & Mount

Create a filesystem (ext4, xfs) and mount to a directory. Now you can read/write data.

4
Modify (Elastic Volumes)

Increase size, change volume type, or increase IOPS — live, without downtime. You cannot decrease size.

5
Detach & Delete

Unmount → Detach → Delete. After deletion, data is gone permanently unless you took a Snapshot.

⚠️

Important: When you terminate an EC2 instance, the root EBS volume is deleted by default. Data volumes are retained unless you explicitly delete them. Always check your delete-on-termination settings!

Section 05

Snapshots — Backups

EBS Snapshots are point-in-time backups of your volumes stored durably in Amazon S3. They are incremental — only changed blocks are saved after the first snapshot.

Analogy: Your first snapshot is a full photocopy of your hard drive. Every subsequent snapshot is just a sticky note saying "here's what changed since last time." Fast and cheap!

AWS CLI — Snapshots 
# Create a snapshot of a volume
aws ec2 create-snapshot \
  --volume-id vol-0abcd1234efgh5678 \
  --description "Daily backup - prod database"

# List all your snapshots
aws ec2 describe-snapshots --owner-ids self

# Restore: create a new volume from a snapshot
aws ec2 create-volume \
  --snapshot-id snap-0abc123def456 \
  --availability-zone us-east-1b

# Copy snapshot to another region (for DR)
aws ec2 copy-snapshot \
  --source-region us-east-1 \
  --source-snapshot-id snap-0abc123def456 \
  --destination-region eu-west-1

Snapshot Best Practices

⏰
Automate with DLM

Use Data Lifecycle Manager to create automated snapshot schedules (daily, weekly).

🌍
Cross-Region Copy

Copy snapshots to another region for disaster recovery and compliance.

🔒
Encrypt Snapshots

Snapshots of encrypted volumes are automatically encrypted. Share with IAM controls.

Section 06

Encryption & Security

EBS encryption uses AWS KMS (Key Management Service) to encrypt data at rest and in transit between EC2 and EBS — transparently, with no performance impact.

What Gets EncryptedWhen
Data at rest on the volumeAlways, for encrypted volumes
Data moving between EC2 and EBSIn transit (encrypted automatically)
Snapshots of the volumeAutomatically encrypted
New volumes from an encrypted snapshotAutomatically encrypted
🔑

Enable Account-Level Encryption: You can enable "Encryption by Default" in your AWS account so that all new EBS volumes are automatically encrypted — recommended for production environments.

Section 07

Pricing Overview

EBS pricing is based on provisioned storage per month — you pay even when the volume is not attached or being used.

Volume TypePrice (us-east-1)Notes
gp3$0.08 / GiB-month3,000 IOPS free; extra IOPS charged
io2$0.125 / GiB-month + $0.065/IOPSHigh-perf, more expensive
st1$0.045 / GiB-monthCheaper for large sequential workloads
sc1$0.015 / GiB-monthCheapest; for cold data
Snapshots$0.05 / GiB-monthIncremental after first
💰

Cost tip: Detached volumes still incur charges. Delete volumes you no longer need, or snapshot and delete them to save money. Use AWS Cost Explorer to monitor EBS spend.
